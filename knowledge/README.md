# knowledge/

Curated, version-controlled notes — the "second brain" of this workspace.

Content here should be:

- **Stable** — things that change rarely and are worth carrying across
  machines and sessions.
- **Readable by agents** — plain markdown, organized so an agent can navigate
  it without prior context.
- **Yours** — preferences, decisions, domain notes, references you return to.

## Structure

There is no required structure. Pick a layout that matches how you actually
look things up. Common starting points:

- `people/` — context on collaborators, teams, organizations.
- `decisions/` — durable decisions and the reasoning behind them.
- `domains/` — notes scoped to a particular subject area.
- `playbooks/` — reusable procedures.

Add directories as you need them. Prune ruthlessly.
